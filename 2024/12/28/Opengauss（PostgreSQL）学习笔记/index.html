可以通过ClassLoader（类加载器）找到properties文件，并导入

类加载器是 Java 的一部分，用于动态地加载类或资源文件（如 .properties 文件）。每个 Java 类或资源文件都是通过类加载器加载到内存中的。

类加载器有三种：

Bootstrap ClassLoader：加载 JDK 的核心类库（如 java.lang.String），无法加载用户自定义的类。(是使用c++编写的，如果使用getClassLoader(),会返回null)

Extension ClassLoader：加载 JDK 的扩展类库（通常位于 JAVA_HOME/lib/ext 下的类）。这种情况一般不常见。

Application ClassLoader：加载 classpath 下的所有类和资源。它是我们自定义类的默认加载器。

类加载器加载顺序：Bootstrap ClassLoader -> Extension ClassLoader -> Application ClassLoader

所以，可以通过类加载器加载properties文件，供后续使用

注：classpath指的是存放class文件的目录，但是在编译后，resources里的properties文件也会被复制到classpath里面

Opengauss & postgreSQL 的内部结构

role和user的区别

权限上的区别：role没有login权限，role + login = user

role相当于 一种抽象的身份，这个身份具有一定的权限，而user是一个实例，将role赋予给user，相当于user继承了role的权限（面向对象的思想），管理员可以通过设置不同的role (read_only,read_write),并将不同的role分配给不同的user，来赋予user权限，便于管理。

实例，database，schema 与 table

postgreSQL实例

一个PostgreSQL实例是一组进程的集合，是一个完整的系统。

database

每个实例可以可以管理多个数据库，每个数据库都是一个独立的逻辑单元。

数据库之间没有直接的共享，不能跨数据库查询（除非使用特定的扩展或配置，如 dblink 或 fdw）。

每个数据库都有自己的数据表、索引、视图、函数、触发器等对象。

作用：

  • 提供数据的物理隔离和逻辑分离。
  • 不同的应用程序或业务单元可以在同一实例上使用不同的数据库，彼此互不干扰。

创建数据库的语法

schema（模式）

模式是数据库中对象的逻辑容器，可以理解为数据库的命名空间。在 PostgreSQL 中，模式用于组织和分隔数据库中的对象，类似于文件系统中的目录。每个数据库中可以有多个模式，一个模式中可以包含多个表、视图、函数等对象。

特性：

  • 命名空间：模式允许你在同一个数据库中有相同名称的表或其他对象，但它们属于不同的模式，因此不会冲突。
  • 默认模式：PostgreSQL 中的默认模式是 public，所有用户创建的对象默认都会放在这个模式中，除非显式指定其他模式。
  • 模式隔离：不同模式中的对象彼此独立，可以通过 schema_name.table_name 的方式访问特定模式中的对象。

作用：

  • 逻辑分离：不同的应用或用户可以使用不同的模式，将它们的表和其他对象隔离开来。
  • 管理方便：可以通过模式来更好地管理权限、访问控制和对象组织。

在 PostgreSQL 中执行查询时，如果你没有指定模式，系统会默认使用 public 模式

table

表是数据库的核心数据存储单元，类似于一个电子表格，其中包含按行和列组织的数据。在 PostgreSQL 中，表存储实际的数据，每一行代表一条记录，每一列代表一个字段或属性。

特性：

  • 行和列：表由行（记录）和列（字段）组成，每一行是一条数据，每一列是一个属性。
  • 主键：表中通常会有一个唯一标识每行数据的列，称为主键（Primary Key），确保数据的唯一性。
  • 数据类型：表的每一列都有特定的数据类型，如 INTEGER, VARCHAR, DATE，这些类型定义了列中可以存储的数据类型。
  • 关系：在关系型数据库中，表与表之间可以通过外键（Foreign Key）建立关联，表示数据之间的关系。

作用：

  • 表是用来存储和组织数据的，数据库的查询操作大多都围绕表进行。
  • 通过定义表，可以清晰描述数据的结构和限制。

基础语句

ROLLUP与CUBE

CUBE：为所有维度的每个组合生成汇总，包括每个字段的全局小计和总体总计。它的计算范围是最广的，提供了更多层次的汇总信息。

ROLLUP：只对从左到右的逐级字段进行汇总，生成的汇总信息相对较少。它不会像 CUBE 一样考虑所有维度的组合。

IN 与 EXSITS

IN 用来判断某个值是否在一组结果中，通常和子查询一起使用，以便从另一张表中筛选数据。IN 可以用于多个具体值，也可以用于子查询的结果集。

要注意：在SQL中NULL表示未知，NOT IN (NULL)，表示的也是未知，所以不会返回任何值，如果要筛选的表里面有NULL，注意使用 WHERE somthing IS NOT NULL

EXISTS 用于判断子查询是否返回结果，简单来说，它只关心子查询是否返回了 至少一行数据，而不关心具体的数据是什么。只要子查询有结果，EXISTS 就会返回 TRUE，否则返回 FALSE。

JOIN的用法

尽量使用JOIN替代旧式逗号连接，显得更加直观,并且显式 JOIN 可能对复杂查询的执行效率有优化效果，但具体性能提升依赖于数据库的查询优化器。在小数据量情况下差别不明显，大量数据时则可能会更优。

(INNER) JOIN

INNER JOIN 是最常见的连接类型，它只返回两个表中满足连接条件的记录。也就是说，只有当两个表中有匹配的记录时才会返回结果。

LEFT JOIN

LEFT JOIN 返回左表的所有记录以及右表中符合连接条件的记录。即使右表中没有匹配的记录，左表的记录仍然会显示，右表未匹配的部分则返回 NULL。

RIGHT JOIN

RIGHT JOIN 与 LEFT JOIN 类似，不过它会返回右表的所有记录和左表中符合条件的记录。即使左表中没有匹配的记录，右表的记录仍然会显示，左表未匹配的部分则返回 NULL。

FULL OUTER JOIN

FULL OUTER JOIN 返回左表和右表的所有记录，无论是否有匹配记录。如果没有匹配，另一侧会返回 NULL。它是 LEFT JOIN 和 RIGHT JOIN 的组合。

CROSS JOIN

CROSS JOIN 返回两个表的笛卡尔积，即每个左表的记录都与右表的每个记录组合。它不需要 ON 子句。

SELF JOIN

SELF JOIN 是将一个表与自身进行连接。通常用于需要在同一表中比较不同行的场景。

权限管理

Opengauss(Postgres)存在一个默认role，public，所有user默认被放在这个role里面。

数据库的所有者没有创建角色的功能。在 PostgreSQL 中，创建角色（用户）的权限只能由具有 超级用户（SUPERUSER） 权限的用户或具有 CREATEROLE 权限的角色执行。

在 PostgreSQL 中，可以使用 GRANT 命令为用户授予访问表的权限。常见的权限包括 SELECT（读取数据）、INSERT（插入数据）、UPDATE（更新数据）和 DELETE（删除数据）等。

PL/pgSQL

PL/pgSQL 是 PostgreSQL 的一种过程语言，具有以下特点：

  • 支持变量声明、控制结构（如 IF、LOOP）、异常处理等。
  • 用于创建存储过程（FUNCTION 或 PROCEDURE）和触发器函数。
  • 主要用来实现数据库内的复杂业务逻辑。

Trigger(触发器)

触发器是一种特殊的数据库对象，当某个表的指定操作（INSERT、UPDATE、DELETE）发生时，自动执行关联的触发器函数。触发器在以下情况下有广泛应用：

  • 维护数据完整性：自动在特定操作时检查或调整数据。
  • 记录日志：对关键操作的日志记录。
  • 级联操作：在一张表的数据变动时，级联更新或删除相关表数据。

触发器的类型

  1. 行级触发器（FOR EACH ROW）：在每一行被影响时触发。
  2. 语句级触发器（FOR EACH STATEMENT）：在整个 SQL 语句执行时触发一次。

常见的触发器时间

  • BEFORE：在操作之前触发。
  • AFTER：在操作之后触发。
  • INSTEAD OF：用于视图，替代默认操作（如 INSERT、UPDATE、DELETE）。
